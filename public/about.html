Lex Fanbot Limitations

The fanbot has access to all transcripts available of the podcast. However there are some episodes for which the list of transcripts are not available. This means that the Fanbot (at the moment) is not capable of searching and creating a response for questions regarding these episodes.

No Memory

Due to the high volume of requests this tool receives and to avoid crashes the system does not keeps track of the full conversation history. In technical terms this means that every question you ask will be treated as the first one without including previous requests in the context. I hope to implement chat context in the near future.

Results

You get "Sorry, I can't help with that" too often

The assistant is instructed to only answer questions about the podcast. Try to rephrase your question to include words "episode", "guest" or "podcast".

Instead of asking "What's the meaning of life?" try "Tell me episodes where they discussed the meaning of life" or "Recommdend episodes about life".

About the project

This is an Open Source initiative that I started by my own because I'm a big fan of the podcast. I don't work for the Lex Fridman Podcast and this runs with personal resources. If you find a bug or problem let me know by opening an issue in the project.

Data Analytics

Google Analytics is in place just to know how people use the fanbot. Additionaly you can see that prompts get stored with the intention to improve the classification of the queries and make the fanbot smarter.
